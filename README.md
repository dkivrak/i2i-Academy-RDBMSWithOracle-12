# i2i Academy - RDBMS With Oracle

This project is a Spring Boot and Oracle Database XE implementation developed for the i2i Academy RDBMS with Oracle assignment.

The application runs with Docker Compose, connects to Oracle XE through JDBC and HikariCP, manages schema creation with Flyway, and uses a PL/SQL package to process book data.

## Technologies

- Java 17
- Spring Boot
- Oracle Database XE
- Oracle JDBC
- HikariCP
- Flyway
- PL/SQL
- Docker & Docker Compose
- DBeaver

## Features

- Oracle XE database deployment with Docker
- Spring Boot application containerized with Docker
- Flyway-based automatic database migration
- Normalized relational schema:
  - AUTHORS
  - PUBLISHERS
  - BOOKS
  - AUDIT_LOGS
- Row-level trigger for auditing book insert operations
- PL/SQL package named BOOK_OPERATIONS
- Raw delimited string parsing into XML and JSON
- Database-side insertion logic through PL/SQL
- Cursor-based book retrieval
- REST API for importing and listing books

## Database Schema

The database contains four normalized main tables.

```text
AUTHORS stores author information.

PUBLISHERS stores publisher information.

BOOKS stores book records and references both AUTHORS and PUBLISHERS using foreign keys.

AUDIT_LOGS stores automatic audit records generated by a row-level trigger whenever a new book is inserted.
```

## PL/SQL Package

```text
The BOOK_OPERATIONS package contains the main database-side logic.

RAW_TO_XML converts raw delimited book data into XML format.

RAW_TO_JSON converts the same raw input into JSON format.

INSERT_BOOKS inserts parsed book records into AUTHORS, PUBLISHERS, and BOOKS tables.

GET_ALL_BOOKS returns complete book details using a SYS_REFCURSOR.
```

## Raw Input Format

The import endpoint expects book data in the following format:

```text
Book Title|Author Name|Publisher Name
```

Multiple books can be separated with semicolons:

```text
Dune|Frank Herbert|Chilton Books;Foundation|Isaac Asimov|Gnome Press
```

## Running the Project

Build the Spring Boot application:

```text
    mvn clean package -DskipTests
```text

Start Oracle XE and the Spring Boot application:

```text
    docker compose up --build
```

Check running containers:

```text
    docker compose ps
```

## API Endpoints

### Import Books

POST /api/books/import

Example:

```text
    curl -X POST http://localhost:8080/api/books/import \
      -H "Content-Type: text/plain" \
      -d "Brave New World|Aldous Huxley|Chatto and Windus"
```

Expected response:

```text
    Books imported successfully.
```

### Get All Books

GET /api/books

Example:

```text
    curl http://localhost:8080/api/books
```

Example response:

```text
    [
      {
        "id": 1,
        "title": "Dune",
        "author": "Frank Herbert",
        "publisher": "Chilton Books"
      },
      {
        "id": 2,
        "title": "Foundation",
        "author": "Isaac Asimov",
        "publisher": "Gnome Press"
      }
    ]
```

## Notes

The Oracle service name used by the application is XEPDB1 because the project runs on Oracle Database XE inside Docker.

Flyway automatically creates the required schema when the Spring Boot application starts.

The BOOKS table trigger automatically writes insert operations into AUDIT_LOGS.

Special XML characters such as & should be avoided in raw input unless XML escaping is handled in the PL/SQL function.

## Assignment Scope

This project demonstrates the integration of relational database design, Oracle PL/SQL, database migration, Docker-based deployment, and Java REST API development in a single working application.
